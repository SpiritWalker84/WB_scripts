# Файлы для коммита в репозиторий

## Структура директории wb_full/

```
wb_full/
├── .gitignore                    # Игнорируемые файлы
├── .env.example                  # Пример конфигурации
├── README.md                     # Основная документация
├── QUICKSTART.md                 # Быстрый старт
├── DEPLOYMENT.md                 # Инструкции по развертыванию
├── GIT_DEPLOY_INSTRUCTIONS.md    # Инструкции по Git
├── CHECKLIST.md                  # Чеклист перед коммитом
├── FILES_TO_COMMIT.md            # Список файлов для коммита
├── requirements.txt              # Зависимости Python
├── check_setup_price.py          # Проверка готовности системы
├── clear_wb_stocks.py            # Удаление остатков
├── download_price.py             # Загрузка прайсов
├── update_wb_stocks_prices.py    # Обновление цен и остатков
├── run_full_update.py            # Единый скрипт
├── *Артикулы*.xlsx              # Файлы соответствия артикулов
└── *Баркоды*.xlsx               # Файлы соответствия баркодов
```

## Файлы, которые НЕ должны быть в коммите

Следующие файлы должны быть в `.gitignore`:

- ❌ `.env` - конфигурация с секретами
- ❌ `wb_key.txt` - API ключи
- ❌ `venv/` - виртуальное окружение
- ❌ `__pycache__/` - кэш Python
- ❌ `*.pyc` - скомпилированные файлы
- ❌ `*.log` - логи

**Примечание:** Файлы с данными (`*.xlsx`, `*.csv`) включены в репозиторий, так как это личный проект.

## Команда для проверки

```bash
# Проверка, что чувствительные файлы игнорируются
git check-ignore -v .env wb_key.txt *.xlsx

# Просмотр статуса
git status

# Просмотр структуры (без игнорируемых файлов)
git ls-files wb_full/
```
